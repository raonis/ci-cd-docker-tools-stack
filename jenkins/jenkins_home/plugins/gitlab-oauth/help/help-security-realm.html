Authentication requires a valid OAuth application to be registered with gitlab.com.
Using your gitlab account you can create a new application registration using this link: https://gitlab.com/settings/applications/new
You can use any name you want but bear in mind this is what gitlab will show to the users when they authenticate and authorize this plugin to act on their behalf.
The entry should look like this:
Main URLhttp://127.0.0.1:8080
Callback URLhttp://127.0.0.1:8080/securityRealm/finishLogin
With 127.0.0.1:8080 replaced with the public hostname and port of your jenkins instance. If the server is private it should be a url that the browser will know how to get to or the IP that the Jenkins instance is being accessed through.

The /securityRealm/finishLogin is required as this is where gitlab will redirect the user for the second part of the autorization process.
